# Repository Instructions

## Repo Shape

- Bun workspaces are `apps/*` and `packages/*`; currently only `apps/desktop` and `packages/database` exist.
- The README mentions `apps/backend`, but that package is not present; trust `package.json`, `turbo.json`, and workspace directories over README prose.
- Use Bun 1.3.11 (`packageManager` and CI). Keep `bun.lock` as the only package lockfile.

## Commands

- Install: `bun install` or CI-equivalent `bun install --frozen-lockfile`.
- Dev all workspaces: `bun run dev`; desktop-only: `bun run --cwd apps/desktop dev`.
- Verification used by CI: `bun run check` then `bun run build`; CI only runs tests if a root `test` script exists, and none exists now.
- Typecheck all packages: `bun run typecheck`; desktop-only: `bun run --cwd apps/desktop typecheck`; database-only: `bun run --cwd packages/database typecheck`.
- Package desktop installers: `bun run build:mac`, `bun run build:win`, `bun run build:linux`, or unpacked `bun run build:unpack`.
- Release helper: `bun run release [major|minor|patch|x.y.z]`; it requires a clean tree, updates only `apps/desktop/package.json`, runs `bun run check` unless `--skip-check`, commits `Release vX.Y.Z`, tags, and pushes unless `--no-push`.

## Tooling

- Lint/format is Ultracite via `oxlint.config.ts` and `oxfmt.config.ts`; `bun run check` checks, `bun run fix` applies fixes.
- Do not manually edit generated/vendor-style files excluded by Ultracite unless the task specifically requires it: `apps/desktop/src/renderer/src/routeTree.gen.ts` and `apps/desktop/src/renderer/src/components/ui/**/*`.
- Pre-commit runs `bun lint` against staged changes and may restage formatted files.

## Desktop App

- Electron entrypoints are `apps/desktop/src/main/index.ts`, `src/preload/index.ts`, and `src/renderer/src/main.tsx`.
- Renderer routing uses TanStack Router with hash history; route files live in `apps/desktop/src/renderer/src/routes`, and `routeTree.gen.ts` is generated by the Vite router plugin.
- Renderer aliases: `@/*` and `@/renderer/*` map to `apps/desktop/src/renderer/src/*`; `@/constants` maps to `apps/desktop/src/constants.ts`.
- App startup is gated through `window.api.app.start()` in `StartupGate`; main startup work is registered in `apps/desktop/src/main/startup.ts` and currently runs database startup before rendering routes.
- Keep Electron IPC APIs exposed through preload modules and `apps/desktop/src/preload/window.d.ts`; renderer code should use `window.api`, not import Electron main/preload internals directly.
- Auto-update is best-effort and disabled in dev/unpackaged builds; Linux self-update only works when running as AppImage (`APPIMAGE` set).

## Database

- `@repo/database` exports only `./client` and `./schemas`; desktop imports it via workspace dependency.
- Schema files live in `packages/database/src/schemas`; after schema changes run `bun run --cwd packages/database generate` to update `packages/database/drizzle` migrations.
- Drizzle uses SQLite with `snake_case` casing. Electron Builder bundles migrations from `packages/database/drizzle` to `resources/database/drizzle`; packaged apps read migrations from `process.resourcesPath`.

## Error Handling

- In backend/non-renderer code, always use `better-result` for expected failures and boundary operations. Prefer typed `Result<T, E>` values, `TaggedError` classes, and `Result.try` / `Result.tryPromise` over throw-based control flow.
- There is a repo-local `better-result-adopt` skill under `.agents/skills`; use it for larger conversions away from try/catch, null sentinels, or thrown domain errors.
